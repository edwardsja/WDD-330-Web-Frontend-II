Week 10 Notes

Validating Forms

Constraint Validation API
element.typeMismatch returns true or false depending if the inserted text matches the input's pattern
element.setCustomValidity(""); used to give custom error messages
setting a custom error message is useful because there is no way to change it's style with CSS, JS can be used, but it requires much more lines of code than just using the constraint validation API

Fetch

init: optional parameter of fetch()
'headers' lets you add additional information with the fetch request
'Content-Type' header allows you to pass what type of file that is being fetched. (ex. 'image/jpeg', or 'application/json')

'redirect' tells the program what to do when the fetch redirects, it can 'follow' the redirect, or throw an 'error' and abort
